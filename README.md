# Pair Programming с AI

Гайд для студентов, которые хотят работать с Cursor, Kimi Code, Claude Code, OpenCode и другими coding-агентами как с полноценным напарником, а не как с кнопкой «сделай за меня».

> **Суть в одном предложении:** ты держишь руль и принимаешь решения, агент помогает писать, проверять и не тупить.

---

## Зачем этот гайд

AI-редакторы уже умеют писать код. Но код, который они пишут без твоего контроля, быстро превращается в лапшу: лишние файлы, странные решения, неочевидные баги. Этот гайд учит дисциплине pair programming — подходу, при котором агент становится умным напарником, а не заменой тебе.

Быстрый старт

1. Открой свой проект в редакторе с AI (Cursor, Kimi Code, Claude Code и т.д.).
2. Опиши задачу одним предложением.
3. Скажи агенту: «Сначала предложи план, потом пишем код».
4. Реализуйте минимальное изменение.
5. Проверь: тесты, diff, здравый смысл.
6. Закоммить с понятным сообщением.

Подробности — в [`docs/04-workflow.md`](docs/04-workflow.md).

## Для кого

- Студенты, которые только начинают кодить и впервые открыли AI-редактор.
- Студенты с опытом, которые хотят перестать воевать с агентом и начать работать вместе.
- Любая среда: Cursor, Kimi Code, Claude Code, OpenCode, GitHub Copilot, Cody.

## Принцип «среда не важна»

Все советы здесь написаны так, чтобы работать в любой среде. Горячие клавиши и названия режимов различаются, но подход — один.

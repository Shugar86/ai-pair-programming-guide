# 02. Агент как pair-programmer

## Ты — не заказчик, а напарник

Легко начать общаться с агентом как с исполнителем: «сделай это», «исправь то». Но в pair programming ты работаешь рядом. Ты не кидаешь задачу и уходишь — ты остаёшься в процессе.

## Твои обязанности

| Что делаешь ты | Почему это важно |
|---|---|
| Формулируешь задачу | Агент не знает, чего ты хочешь, пока ты не скажешь |
| Выбираешь приоритеты | Время и ресурсы ограничены |
| Решаешь компромиссы | «Быстро» vs «красиво» — только ты знаешь, что важнее |
| Проверяешь diff | Агент может навредить, не заметив |
| Запускаешь тесты | Автотесты — твоя страховка |
| Задаёшь направление | Агент предлагает, ты решаешь |

## Обязанности агента

| Что делает агент | Почему это помогает |
|---|---|
| Пишет код | Освобождает тебя от рутины |
| Предлагает план | Помогает увидеть структуру до кода |
| Ищет ошибки | Свежий взгляд на твой код |
| Запускает команды | Ускоряет проверку |
| Задаёт уточняющие вопросы | Сужает неопределённость |

## Где граница

**Ты не делегируешь:**
- Понимание задачи.
- Решение об архитектуре.
- Выбор между несколькими равными вариантами.
- Ответственность за результат.

**Агент не должен:**
- Делать большие изменения без согласования.
- Пушить в прод без твоей проверки.
- Притворяться, что всё понял, если не понял.

## Когда ты ведёшь, а когда агент

```
Ты:         "Нужно добавить сортировку по дате"
Агент:      "Предлагаю план: 1) добавить поле, 2) обновить запрос, 3) обновить UI"
Ты:         "Ок, начни с запроса"
Агент:      [пишет код]
Ты:         [проверяет diff]
Ты:         "Здесь лучше использовать индекс, а не сортировку в памяти"
Агент:      [исправляет]
```

## Практика

Перед каждым запросом задай себе вопрос: «Я сейчас отдаю решение агенту или работаю вместе с ним?» Если отдаёшь — верни себе руль.

# 02. Агент как pair-programmer

## Ты — не заказчик, а напарник

Легко начать общаться с агентом как с исполнителем: «сделай это», «исправь то». Но в pair programming ты работаешь рядом. Ты не кидаешь задачу и уходишь — ты остаёшься в процессе.

Пример «плохого» взаимодействия:

```text
Ты: "Сделай мне авторизацию"
AI: [пишет кучу кода, тянет библиотеки, меняет роутинг]
Ты: [жмёшь Accept all, не читая]
```

Пример нормального взаимодействия:

```text
Ты: "Нужно добавить простую авторизацию только для админ-панели.
 Один пользователь, логин/пароль можно задать в .env."
AI: "Давай сначала план в 3 шага, потом код"
Ты: "Ок, но не трогай публичные роуты и не добавляй регистрацию"
AI: [даёт план]
Ты: [корректируешь план, потом уже код]
```


## Твои обязанности

| Что делаешь ты | Почему это важно |
| :-- | :-- |
| Формулируешь задачу | Агент не знает, чего ты хочешь, пока ты не скажешь |
| Выбираешь приоритеты | Время и ресурсы ограничены |
| Решаешь компромиссы | «Быстро» vs «красиво» — только ты знаешь, что важнее |
| Проверяешь diff | Агент может навредить, не заметив |
| Запускаешь тесты | Автотесты — твоя страховка |
| Задаёшь направление | Агент предлагает, ты решаешь |
| Обновляешь документацию | Только ты знаешь, что нужно следующим людям |

Хороший вопрос, который стоит задавать себе каждые 10–15 минут:
«Я сейчас управляю процессом или просто жму кнопки?»

## Обязанности агента

| Что делает агент | Почему это помогает |
| :-- | :-- |
| Пишет код | Освобождает тебя от рутины |
| Предлагает план | Помогает увидеть структуру до кода |
| Ищет ошибки | Свежий взгляд на твой код |
| Запускает команды | Ускоряет проверку |
| Задаёт уточняющие вопросы | Сужает неопределённость |
| Генерирует тесты и фикстуры | Помогает закрепить поведение изменения |

Ты можешь явно задавать «роль» агента в промпте: ревьюер, джун, архитектор, тестировщик.

Пример:

```text
Сейчас ты выступаешь как строгий ревьюер.
Не пиши код. Только найди проблемы в этом diff и предложи, как упростить.
```


## Где граница

**Ты не делегируешь:**

- Понимание задачи.
- Решение об архитектуре.
- Выбор между несколькими равными вариантами.
- Ответственность за результат.

**Агент не должен:**

- Делать большие изменения без согласования.
- Пушить в прод без твоей проверки.
- Притворяться, что всё понял, если не понял.

Если чувствуешь, что агент «таскает тебя за собой» — вернись к исходной постановке задачи и переформулируй, чего ты хочешь добиться.

## Когда ты ведёшь, а когда агент

```text
Ты: "Нужно добавить сортировку по дате"
Агент: "Предлагаю план: 1) добавить поле, 2) обновить запрос, 3) обновить UI"
Ты: "Ок, начни с запроса. Остальное пока не трогай"
Агент: [пишет код]
Ты: [проверяешь diff]
Ты: "Здесь лучше использовать индекс, а не сортировку в памяти"
Агент: [исправляет]
Ты: [запускаешь тесты]
```


## Практика

Перед каждым запросом задай себе вопрос:
«Я сейчас отдаю решение агенту или работаю вместе с ним?»
Если отдаёшь — верни себе руль.

Попробуй хотя бы одну сессию провести с установкой:
«AI — джун. Я — сеньор, который отвечает за всё».
Это быстро лечит от желания нажимать «сделай всё сам».

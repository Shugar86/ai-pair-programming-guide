# 02. Агент как pair-programmer

← [01. Что такое pair programming](01-what-is-pair-programming.md) | [03. Принципы](03-principles.md) →

Совместное создание — главная идея. Ты не кидаешь задачу агенту и не уходишь. Ты остаёшься в диалоге, выбираешь направление и несёшь ответственность. Агент — не исполнитель, а напарник, который берёт рутину и предлагает варианты.

## Ты — не заказчик, а напарник

Плохой тон:

```text
Ты: Сделай мне авторизацию.
Агент: [пишет кучу кода, тянет библиотеки, меняет роутинг]
Ты: [жмёшь Accept all, не читая]
```

Нормальный тон:

```text
Ты: Нужно добавить простую авторизацию только для админ-панели. Один пользователь, логин/пароль задаются в .env.
Агент: Сначала план в три шага, потом код?
Ты: Да. Не трогай публичные роуты и не добавляй регистрацию.
Агент: [даёт план]
Ты: [корректируешь план, потом разрешаешь код]
```

Разница в одном: во втором случае ты остаёшься в процессе.

## Как делим работу

| Что делаешь ты | Зачем |
| :-- | :-- |
| Формулируешь задачу | Агент не угадает, чего ты хочешь |
| Выбираешь приоритеты | Время и ресурсы ограничены |
| Решаешь компромиссы | Только ты знаешь, что важнее сейчас |
| Проверяешь diff | Агент может навредить, не заметив |
| Запускаешь тесты | Автотесты — твоя страховка |
| Задаёшь направление | Агент предлагает, ты решаешь |
| Обновляешь документацию | Только ты знаешь, что нужно следующим людям |

| Что делает агент | Зачем |
| :-- | :-- |
| Пишет код | Берёт рутину |
| Предлагает план | Показывает структуру до кода |
| Ищет ошибки | Свежий взгляд |
| Запускает команды | Ускоряет проверку |
| Задаёт уточняющие вопросы | Сужает неопределённость |
| Генерирует тесты и фикстуры | Закрепляет поведение |
| Говорит, когда не уверен | Сужает зону риска |

## Когда ты ведёшь, а когда агент

Ты ведёшь, когда решаешь, что делать и зачем. Агент ведёт, когда ищет варианты реализации и пишет код. Переключение происходит каждые пару минут.

```text
Ты: Нужно добавить кнопку «Копировать ссылку» в карточку заказа.
Агент: План: добавить компонент кнопки, обработчик копирования, сообщение об успехе.
Ты: Ок, начни с обработчика. Сообщение пока не трогаем.
Агент: [пишет код]
Ты: [проверяешь diff]
Ты: Здесь лучше использовать navigator.clipboard, а не execCommand.
Агент: [исправляет]
Ты: [запускаешь тесты]
```

Здесь ты задаёшь scope, агент выполняет, ты корректируешь.

## Границы

**Ты не делегируешь:**

- понимание задачи;
- решение об архитектуре;
- выбор между равными вариантами;
- ответственность за результат.

**Агент не должен:**

- делать большие изменения без согласования;
- применять правки без твоей проверки;
- притворяться, что всё понял, если не понял.

Если чувствуешь, что агент тащит тебя за собой, остановись. Закрой чат, посмотри diff, сформулируй своими словами, что происходит, и вернись к задаче.

## За пределами одного агента

Скоро в работе будет не один агент, а несколько: один пишет код, другой ревьюит, третий ищет баги, четвёртый ведёт документацию. Сейчас ты можешь имитировать это, меняя роль агента в промпте.

```text
Сейчас ты — строгий ревьюер. Не пиши код. Найди проблемы в diff и предложи, как упростить.
```

Когда multi-agent станет обычным делом, правила не изменятся: ты всё равно держишь направление и проверяешь результат. Подробнее — в [`11-advanced-scenarios.md`](11-advanced-scenarios.md).

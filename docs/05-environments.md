# 05. Среды

Принципы одинаковы везде. Различаются только интерфейсы, горячие клавиши и названия режимов. Ниже — краткие карточки для популярных сред.

---

## Cursor

**Особенности:**

- Режим **Chat** — общение с агентом в боковой панели.
- Режим **Composer / Agent** — агент может редактировать несколько файлов и запускать команды.
- **Plan mode** — сначала план, потом код.
- `.cursorrules` — файл с правилами поведения агента в проекте.

**Как начать:**

1. Открой проект в Cursor.
2. Нажми `Cmd/Ctrl + L` — откроется чат.
3. Опиши задачу и попроси план.
4. В режиме Agent дай разрешение на команды.

**Что помнить:** Cursor любит делать много изменений сразу. Контролируй область задачи (scope): смотри на diff, используй checkpoints перед крупными изменениями.

---

## Kimi Code

**Особенности:**

- Работает в терминале и IDE.
- Поддерживает **skills** — переиспользуемые инструкции.
- Читает `AGENTS.md` в корне проекта.
- Имеет доступ к bash, файлам, поиску.

**Как начать:**

1. Установи среду по официальной инструкции.
2. Зайди в папку проекта.
3. Создай `AGENTS.md` с правилами (шаблон в [`templates/AGENTS.md`](../templates/AGENTS.md)).
4. Запусти клиент и опиши задачу.

**Что помнить:** Kimi уважает `AGENTS.md`. Если правила не написаны — агент угадывает. Потрать 10 минут на хороший `AGENTS.md`, сэкономишь часы на объяснениях.

---

## Claude Code

**Особенности:**

- Терминальный интерфейс.
- Мощные инструменты: bash, edit, view, grep.
- Хорошо работает с большими проектами.
- Естественный разговорный стиль.

**Как начать:**

1. Установи Claude Code.
2. Запусти `claude` в папке проекта.
3. Опиши задачу текстом.
4. Используй команды `/help` и `/clear`.

**Что помнить:** Claude Code любит обсуждать. Не бойся просить «короче» или «просто сделай». Чередуй режимы «обсудили → применили → проверили», а не держи всё в бесконечном чате.

---

## OpenCode

**Особенности:**

- Открытый AI-ассистент для программирования.
- Интегрируется с редакторами и CLI.
- Поддерживает MCP (Model Context Protocol) для расширения инструментов.
- Поддержка нескольких провайдеров: OpenAI, Anthropic, Ollama и другие.
- Настраивается через `opencode.json`.

**Как начать:**

1. Установи OpenCode по официальной инструкции.
2. Настрой провайдера модели через `opencode.json`.
3. Открой проект и начни диалог.

**Что помнить:** Конфигурация гибкая, но требует внимания. Начни с простого сетапа, а расширенные инструменты (дополнительные провайдеры моделей через MCP) подключай, когда уже освоишь базу.

---

## GitHub Copilot / Cody

**Особенности:**

- Copilot: встроен в VS Code, IDE JetBrains, Neovim.
- Cody (Sourcegraph): чат, команды, код-интеллект.
- Оба хороши для автодополнения и встроенного чата.

**Как начать:**

1. Установи расширение.
2. Авторизуйся.
3. Используй чат для вопросов и inline suggestions для мелких правок.

**Что помнить:** Copilot/Cody — скорее помощник в моменте. Для крупных задач используй полноценные агенты (Cursor, Kimi, Claude). Не пытайся через чистый автокомплит переписать полпроекта.

# Пример: рефакторинг в Claude Code

**Среда:** Claude Code (терминал)  
**Задача:** Вынести повторяющуюся логику валидации из трёх handler'ов в отдельную функцию.

---

## 1. Определяем задачу

```text
Ты: В трёх handler'ах дублируется проверка входных данных. Вынеси её в отдельную функцию.
```

## 2. Согласуем план

```text
Агент: План:
1. Найти три handler'а с дублирующейся валидацией.
2. Создать функцию validate_input в validators.py.
3. Заменить дублирующий код на вызов validate_input.
4. Запустить тесты.

Ты: Ок, но не меняй логику валидации — только перенеси.
```

## 3. Реализация

```text
Агент: [показывает diff]
Ты: [видит, что функция называется validate_user_input, хотя валидируются не только пользователи]
Ты: Переименуй в validate_payload, это универсальнее.
Агент: [переименовывает]
```

## 4. Проверка

```text
Агент: pytest tests/
Ты: [проверяет, что нет изменений в поведении]
```

## 5. Коммит

```bash
git add app/validators.py app/handlers/
git commit -m "refactor: extract validate_payload helper"
```

## 6. Рефлексия

- Сработало: ограничение «только перенеси» удержало scope.
- Можно лучше: сразу дать ожидаемое имя функции.

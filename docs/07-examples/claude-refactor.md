# Пример: рефакторинг в Claude Code

**Вердикт:** рефакторинг без тестов — это рулетка.

**Среда:** Claude Code (терминал)  
**Задача:** Вынести повторяющуюся логику валидации из трёх handler'ов в отдельную функцию.

---

## 1. Определяем задачу

```text
Ты: В трёх handler'ах дублируется проверка входных данных. Вынеси её в отдельную функцию.
     Не меняй логику валидации — только перенеси.
     Используй имя validate_payload, потому что валидируются не только пользователи.
```

## 2. Согласуем план

```text
Агент: План:
1. Найти три handler'а с дублирующейся валидацией.
2. Создать функцию validate_payload в validators.py.
3. Заменить дублирующий код на вызов validate_payload.
4. Запустить тесты.

Ты: Ок. Если тестов на эту валидацию нет — добавь один минимальный.
```

## 3. Реализация и тупик

Агент показывает diff. Ты замечаешь, что функция называется validate_user_input, хотя валидируются и заказы, и платежи.

```text
Ты: Переименуй в validate_payload, как договаривались.
Агент: [переименовывает]
```

Агент запускает тесты. Один падает.

```text
Агент: pytest tests/
Агент: FAILED tests/handlers/test_payment.py::test_missing_field
Агент: Ошибка: validate_payload не принимает необязательный параметр strict, который был в одном из handler'ов.
Ты: Верни старый вызов в payment handler и оставь strict на месте. Перенеси только общую часть.
Агент: [возвращает strict-логику в payment handler]
```

## 4. Проверка

```text
Агент: pytest tests/
Агент: Все тесты прошли.
Ты: [проверяет diff] Хорошо. Убедись, что поведение в трёх handler'ах не изменилось.
Агент: [запускает тесты ещё раз и показывает вывод]
```

## 5. Коммит

```bash
git add app/validators.py app/handlers/
git commit -m "refactor: extract validate_payload helper"
```

## 6. Рефлексия

- Сработало: ограничение «только перенеси» удержало scope. Имя функции заранее сократило один цикл правок.
- Тупик: необязательный параметр strict оказался только в одном handler'е. Агент пытался унифицировать то, что унифицировать нельзя.
- Можно лучше: сразу показать агенту примеры вызовов из всех трёх handler'ов, чтобы он видел различия.

## Мини-вывод

Рефакторинг — не про идеальную симметрию, а про сохранение поведения. Если разные пути имеют право отличаться — скажи об этом явно. Claude хорошо исправляет направление после конкретного фидбека.

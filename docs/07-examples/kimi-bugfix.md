# Пример: починка бага в Kimi Code

**Среда:** Kimi Code (CLI)  
**Задача:** API возвращает 500 при создании заказа без адреса.

---

## 1. Определяем задачу

```text
Ты: При создании заказа без адреса API падает с 500. Нужно вернуть 400 с понятной ошибкой.
```

## 2. Согласуем план

```text
Агент: План:
1. Найти endpoint создания заказа.
2. Добавить валидацию адреса.
3. Вернуть 400, если адрес отсутствует.
4. Добавить тест на этот случай.

Ты: Ок. Начни с теста, чтобы воспроизвести ошибку.
```

## 3. Реализация

```text
Агент: [создаёт тест, показывает, что он падает с 500]
Ты: Хорошо, теперь почини валидацию.
Агент: [добавляет проверку в handler]
Ты: [смотрит diff] Почему ты изменил OrderService? Задача была только про endpoint.
Агент: [переносит проверку в endpoint]
```

## 4. Проверка

```text
Агент: pytest tests/orders/test_create.py -v
Ты: [видит, что тест проходит]
```

## 5. Коммит

```bash
git add tests/orders/test_create.py app/orders/routes.py
git commit -m "fix: return 400 when order address is missing"
```

## 6. Рефлексия

- Сработало: тест сначала помог увидеть баг до фикса.
- Можно лучше: сразу уточнить, где должна быть валидация.

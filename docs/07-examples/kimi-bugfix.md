# Пример: починка бага в Kimi Code

**Среда:** Kimi Code (CLI)  
**Задача:** API возвращает 500 при создании заказа без адреса.

---

## 1. Определяем задачу

```text
Ты: При создании заказа без адреса API падает с 500. Нужно вернуть 400 с понятной ошибкой.
     Валидация должна жить в endpoint, не в сервисе.
     Начни с теста, который воспроизводит баг.
```

## 2. Согласуем план

```text
Агент: План:
1. Найти endpoint создания заказа.
2. Написать тест, который шлёт запрос без адреса и ожидает 400.
3. Добавить проверку адреса в endpoint.
4. Запустить тест и убедиться, что он проходит.

Ты: Ок. Только endpoint — не лезь в OrderService.
```

## 3. Реализация и тупик

Агент создаёт тест. Тест падает с 500 — баг воспроизведён.

```text
Ты: Хорошо, теперь почини валидацию в endpoint.
Агент: [добавляет проверку в app/orders/routes.py]
```

Ты смотришь diff и видишь, что агент изменил OrderService, добавив туда валидацию.

```text
Ты: Убери изменения из OrderService. Валидация должна быть в endpoint, как договаривались.
Агент: [переносит проверку в endpoint]
```

Агент переносит, но теперь endpoint возвращает общее сообщение «Invalid input» вместо понятного «address is required».

```text
Ты: Сообщение об ошибке должно быть конкретным: "address is required".
Агент: [исправляет сообщение]
```

## 4. Проверка

```text
Агент: pytest tests/orders/test_create.py -v
Агент: tests/orders/test_create.py::test_create_order_without_address PASSED
Агент: tests/orders/test_create.py::test_create_order_with_address PASSED
Ты: [видит, что новый тест проходит, а старые не сломались]
```

## 5. Коммит

```bash
git add tests/orders/test_create.py app/orders/routes.py
git commit -m "fix: return 400 when order address is missing"
```

## 6. Рефлексия

- Сработало: тест сначала зафиксировал баг до фикса. Ограничение «только endpoint» помогло удержать архитектуру.
- Первый тупик: агент перенёс валидацию в сервис, потому что так «чище». Ты вернул его к договорённости.
- Второй тупик: общее сообщение об ошибке. Конкретика в ошибке — часть задачи, и её стоит озвучивать сразу.
- Можно лучше: сразу дать пример ожидаемого JSON с ошибкой.

## Мини-вывод

Баг-фикс — это не только починка, но и защита от повторения. Тест сначала даёт тебе рычаг: ты видишь баг, прежде чем чинишь. Kimi в CLI быстро откатывает лишние изменения, если ты чётко держишь границу.

Какие паттерны здесь работали — см. [`06-prompt-patterns.md`](../06-prompt-patterns.md).

# 08. Упражнения

Практика важнее теории. Возьми свой учебный проект или заведи новый репозиторий и пройди эти упражнения.

---

## Упражнение 1. Минимальный diff

**Задача:** изменить цвет кнопки с синего на зелёный.

**Что делать:**
1. Попроси агента сделать изменение.
2. Проверь diff.
3. Если агент зацепил лишние файлы — откати и попроси сделать только нужное.

**Критерий успеха:** в diff ровно один файл и одна строка.

---

## Упражнение 2. План до кода

**Задача:** добавить форму обратной связи на страницу контактов.

**Что делать:**
1. Запрети агенту писать код, пока вы не согласуете план.
2. Согласуй план из 4-5 шагов.
3. Реализуйте только первый шаг.

**Критерий успеха:** есть письменный план и один рабочий коммит.

---

## Упражнение 3. Ревью чужого diff'а

**Задача:** найди в своём проекте последний коммит, сделанный с помощью агента.

**Что делать:**
1. Открой diff коммита.
2. Найди 3 вещи, которые можно было бы сделать лучше (лишний код, странное имя, отсутствие тестов).
3. Создай follow-up коммит с исправлениями.

**Критерий успеха:** три конкретных замечания и один follow-up коммит.

---

## Упражнение 4. Напиши AGENTS.md

**Задача:** создай `AGENTS.md` для своего проекта.

**Что делать:**
1. Возьми шаблон из [`templates/AGENTS.md`](../templates/AGENTS.md).
2. Заполни: язык, стек, правила коммитов, запреты.
3. Проведи одну сессию с агентом, следуя этому контракту.

**Критерий успеха:** файл `AGENTS.md` в репозитории и один коммит, сделанный по его правилам.

---

## Упражнение 5. Рефакторинг под контролем

**Задача:** вынести дублирующийся код в отдельную функцию.

**Что делать:**
1. Найди 2-3 места с одинаковой логикой.
2. Согласуй с агентом имя новой функции и место.
3. Сделай рефакторинг.
4. Запусти тесты.

**Критерий успеха:** тесты проходят, diff только по делу.

# 09. Чеклисты

Копируй эти чеклисты в свои проекты или распечатывай — как удобнее.

---

## Перед началом сессии

- [ ] Ясно сформулирована задача одним предложением.
- [ ] Я понимаю, зачем это нужно (ценность/ставка).
- [ ] Я знаю, как проверить результат.
- [ ] Я не пытаюсь решить 3 задачи одновременно.

---

## Перед принятием изменений агента

- [ ] Я прочитал весь diff.
- [ ] Каждое изменение относится к текущей задаче.
- [ ] Нет кода "на будущее".
- [ ] Имена переменных/функций понятные.
- [ ] Нет хардкода секретов и паролей.
- [ ] Я понимаю, почему агент сделал именно так.

---

## Перед коммитом

- [ ] Тесты проходят (или есть причина, почему их нет).
- [ ] Lint/formatting не ругается.
- [ ] Сообщение коммита описывает суть изменения.
- [ ] В коммит не попали `.env`, ключи, локальные настройки.
- [ ] Diff соответствует сообщению коммита.

---

## Перед показом кода другим

- [ ] Код решает задачу, а не просто "работает".
- [ ] Я могу объяснить каждое изменение.
- [ ] Я удалил отладочные print/console.log.
- [ ] Документация/комментарии обновлены, если нужно.
- [ ] Я сам бы принял этот код на ревью.

---

## В формате для копирования

```markdown
### Перед коммитом

- [ ] Тесты проходят
- [ ] Lint/formatting чистый
- [ ] Сообщение коммита описывает суть
- [ ] Без секретов и `.env`
- [ ] Diff соответствует сообщению
```

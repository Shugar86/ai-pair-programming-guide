# 12. Установка и настройка окружения

Чтобы применять принципы pair programming с AI, нужен редактор с агентом, git и базовый стек проекта. Здесь — пошаговый чеклист для старта.

---

## Что понадобится

1. **Компьютер** с Windows, macOS или Linux.
2. **Редактор кода с AI-агентом** — Cursor, Claude Code, OpenCode, Copilot и т.д.
3. **Git** — система контроля версий.
4. **Аккаунт на GitHub** — для хранения и публикации кода.
5. **Интерпретатор/компилятор** под твой стек: Node.js, Python, Java, Go и т.д.

---

## Шаг 1. Редактор с AI-агентом

Выбирай любой. Все принципы гайда работают в любой среде.

### Cursor

- **Что это:** IDE на базе VS Code с встроенным AI-агентом.
- **Где скачать:** https://www.cursor.com/
- **Что нужно:** аккаунт Cursor, можно авторизоваться через GitHub.
- **После установки:** открой свой проект, нажми `Cmd/Ctrl + L` — откроется чат.
- **Стоимость:** есть бесплатный тариф с лимитами; платный — от $20/мес.

### Claude Code

- **Что это:** терминальный AI-ассистент от Anthropic.
- **Где скачать:** https://claude.ai/code
- **Что нужно:** аккаунт Anthropic, API-ключ или подписка Pro.
- **После установки:** запусти `claude` в папке проекта.
- **Особенность:** работает прямо в терминале, хорош для больших проектов.

### Kimi Code

- **Что это:** AI-кодинг ассистент от Moonshot AI.
- **Где скачать:** https://kimi.moonshot.cn/ (раздел для разработчиков / IDE-плагин).
- **Что нужно:** аккаунт Moonshot AI.
- **После установки:** плагин в IDE или отдельный клиент по инструкции.
- **Примечание:** точный формат клиента может меняться — смотри актуальную документацию.

### OpenCode

- **Что это:** открытый AI-кодинг ассистент с поддержкой нескольких провайдеров.
- **Где скачать:** https://opencode.ai/
- **Что нужно:** установить сам OpenCode и настроить провайдера (OpenAI, Anthropic, Ollama и др.) через `opencode.json`.
- **После установки:** запусти в папке проекта и начни диалог.
- **Плюс:** можно использовать собственные API-ключи и локальные модели.

### GitHub Copilot

- **Что это:** AI-ассистент от GitHub, встроенный в популярные редакторы.
- **Где скачать:** плагин для VS Code / JetBrains / Neovim.
- **Что нужно:** аккаунт GitHub и подписка Copilot (есть бесплатный пробный период для студентов).
- **После установки:** авторизуй плагин и пиши код — ассистент будет предлагать продолжения.

### Cody (Sourcegraph)

- **Что это:** AI-ассистент с продвинутым поиском по коду.
- **Где скачать:** https://sourcegraph.com/cody
- **Что нужно:** аккаунт Sourcegraph, плагин для редактора.
- **После установки:** используй чат и команды для работы с кодом.

## Шаг 2. Git

Git нужен, чтобы фиксировать изменения, экспериментировать безопасно и публиковать код.

### Установка

- **Windows:** скачай установщик с https://git-scm.com/download/win
- **macOS:** `xcode-select --install` или установи через Homebrew: `brew install git`
- **Linux:** `sudo apt install git` (Debian/Ubuntu) или `sudo dnf install git` (Fedora)

### Проверка

```bash
git --version
```

Должно вывести что-то вроде `git version 2.43.0`.

### Базовая настройка

```bash
git config --global user.name "Твоё Имя"
git config --global user.email "you@example.com"
```

### Если что-то пошло не так

- **Установщик не запускается** — скачай версию для своей разрядности (64-bit для современных компьютеров).
- **`git` не recognized** — перезагрузи терминал или компьютер после установки.
- **На Windows лучше использовать Git Bash** — он идёт в комплекте с установщиком git.

Больше про типичные проблемы — в [`15-troubleshooting.md`](15-troubleshooting.md).

## Шаг 3. GitHub

GitHub — это облачное хранилище для git-репозиториев.

1. Зарегистрируйся: https://github.com/signup
2. Установи GitHub CLI (опционально, но удобно):
   - **Windows/macOS/Linux:** https://cli.github.com/
   - Проверка: `gh --version`
3. Авторизуйся: `gh auth login`

## Шаг 4. Стек проекта

Установи то, на чём пишешь:

- **JavaScript/TypeScript:** Node.js с https://nodejs.org/ (рекомендуется LTS).
- **Python:** https://www.python.org/downloads/
- **Java:** JDK (Eclipse Temurin, Amazon Corretto или другой).
- **Go:** https://go.dev/dl/
- **Rust:** https://www.rust-lang.org/tools/install

Проверь, что всё работает:

```bash
node --version
python3 --version
```

## Шаг 5. Пакеты и зависимости проекта

Почти любой проект использует библиотеки. Вот как их устанавливать.

### JavaScript / TypeScript

```bash
npm install      # если в проекте package.json
# или
yarn install
# или
pnpm install
```

### Python

```bash
pip install -r requirements.txt
# или
poetry install
```

### Rust

```bash
cargo build
```

### Go

```bash
go mod download
```

### Системные пакеты (Linux)

```bash
sudo apt update && sudo apt install <название-пакета>   # Debian/Ubuntu
sudo dnf install <название-пакета>                      # Fedora
brew install <название-пакета>                          # macOS
```

Если команда выдаёт ошибку — сначала проверь, что нужный менеджер пакетов установлен. Подробнее — в [`15-troubleshooting.md`](15-troubleshooting.md).

## Шаг 6. Проверка готовности

После установки у тебя должно быть:

- [ ] Редактор с AI-агентом запускается.
- [ ] `git --version` показывает версию.
- [ ] Аккаунт на GitHub создан.
- [ ] Стек проекта установлен (Node/Python/Go и т.д.).
- [ ] Пакеты проекта устанавливаются без ошибок.

Если всё на месте — можно открывать проект и пробовать первую сессию pair programming.

# 14. Рабочий день: IDE или терминал

🌐 [English](en/14-ide-cli-workflow.md) · **Русский**

IDE и CLI — два рабочих стола. IDE лучше, когда хочешь видеть. CLI — когда хочешь двигаться быстро или работаешь на сервере.

---

## IDE vs CLI: когда что

| Ситуация | IDE (Cursor, VS Code + Copilot) | CLI (Claude Code, OpenCode) |
|---|---|---|
| Визуальный diff, кнопки, навигация | ✅ | ❌ |
| Быстрая правка одного файла | ✅ | ✅ |
| Большой проект, много файлов | ✅ | ✅ |
| Работа на удалённом сервере (SSH/VDS) | ❌ | ✅ |
| Хочется минимум интерфейса | ❌ | ✅ |
| Первый опыт с агентом | ✅ | ❌ |

Не знаешь, с чего начать — бери IDE. Там проще увидеть, что агент меняет.

**Диалог: что выбрать сегодня**

```text
Ты: Нужно быстро поправить один конфиг на сервере. IDE или CLI?
Агент: CLI. Открой терминал, подключись по SSH и запусти агента прямо там. IDE для сервера неудобна.
Ты: А если я новичок и боюсь терминала?
Агент: Тогда начни с IDE. Главное — открыть папку проекта и смотреть diff перед любыми изменениями.
```

---

## Типичный рабочий день в IDE

### 1. Открой проект

`File → Open Folder...` и выбери папку проекта. Открыта должна быть именно папка, а не отдельный файл.

### 2. Открой чат с агентом

- **Cursor:** `Cmd/Ctrl + L` — боковая панель чата.
- **VS Code + Copilot:** иконка Copilot в боковой панели.
- **Kimi Code / другие плагины:** своя иконка в sidebar.

### 3. Задай задачу

Напиши одно предложение и попроси план:

```text
Нужно добавить валидацию email при регистрации.
Сначала предложи план, потом пишем код.
```

### 4. Смотри diff

Агент предложит изменения. Не жми сразу «Accept all»:

- открой diff (вкладка **Diff** или иконка изменений);
- прочитай, что изменилось;
- если агент зацепил лишние файлы — скажи об этом.

### 5. Запусти проверки

Встроенный терминал: `Cmd/Ctrl + `` ` (обратный апостроф).

```bash
npm test
# или
pytest
```

### 6. Закоммить

```bash
git add src/auth/validation.py
git commit -m "feat: add email validation in signup"
```

## Типичный рабочий день в CLI

CLI-шаги — это та же петля из [`04-workflow.md`](04-workflow.md), только в терминале.

### 1. Зайди в папку проекта

```bash
cd ~/projects/my-app
```

### 2. Запусти агента

```bash
claude
# или
kimi
# или
opencode
```

### 3. Опиши задачу

```text
Нужно добавить валидацию email при регистрации.
Сначала предложи план, потом пишем код.
```

Это шаг 1 «Захвати идею» и шаг 2 «Наметь путь» из петли 04.

### 4. Контролируй изменения

Агент будет предлагать правки. Перед согласием:

```bash
git diff
```

Это шаг 4 «Проверь на ощупь»: diff перед тестами.

### 5. Запусти проверки

```bash
npm test
```

Это всё ещё шаг 4: тесты, линтер, типы. Список команд проекта — в [`05-environments.md`](05-environments.md).

### 6. Закоммить

```bash
git add src/auth/validation.py
git commit -m "feat: add email validation in signup"
```

Это шаг 5 «Закрепи». Push остаётся за тобой — см. «AI и Git» в [`13-git-github.md`](13-git-github.md).

### 7. Рефлексия

После коммита — две-три строки: что сработало, что можно было быстрее, что в следующий раз. Это шаг 6 «Посмотри, что получилось».

## Связь CLI-шагов с петлёй

| CLI-шаг | Шаг петли 04 | Что проверяешь |
| :-- | :-- | :-- |
| Описал задачу и попросил план | 1–2. Захвати идею, Наметь путь | Идея ясная, план короткий |
| Агент пишет код | 3. Слепи минимум | Только нужные файлы |
| `git diff` | 4. Проверь на ощупь | Нет лишнего, нет странного |
| `npm test` / `pytest` | 4. Проверь на ощупь | Код работает |
| `git commit` | 5. Закрепи | История понятная |
| Короткая ретро-заметка | 6. Посмотри, что получилось | Что улучшить в следующий раз |

## Как показать код агенту

Агенту нужен контекст. Покажи конкретные файлы или фрагменты.

### В IDE

- **`@file` или `#file`** — упомяни файл в чате: `@src/auth/validation.py`.
- **Выдели код мышкой** — многие агенты видят выделенный фрагмент.
- **Скопируй в чат** — для коротких кусков.

### В CLI

- Назови файл текстом: «Посмотри `src/auth/validation.py`».
- Для сложных случаев используй `AGENTS.md` в корне проекта — агент прочитает его сам.

| Действие | Windows/Linux | macOS |
|---|---|---|
| Командная палитра | `Ctrl + Shift + P` | `Cmd + Shift + P` |
| Терминал | `` Ctrl + ` `` | `` Cmd + ` `` |
| Найти файл | `Ctrl + P` | `Cmd + P` |
| Найти по проекту | `Ctrl + Shift + F` | `Cmd + Shift + F` |
| Отменить последнее действие | `Ctrl + Z` | `Cmd + Z` |

## Как организовать окно

Удобный минималистичный layout:

- **Слева** — файлы проекта.
- **По центру** — код.
- **Справа** — чат с агентом.
- **Снизу** — терминал.

Ты должен видеть и код, и то, что пишет агент, без постоянного переключения.

---

## Когда переключаться с IDE на CLI

- Работаешь на сервере по SSH.
- Проект большой, и IDE тормозит.
- Хочешь быстро проверить что-то без открытия редактора.
- Ты уже хорошо знаешь терминал.

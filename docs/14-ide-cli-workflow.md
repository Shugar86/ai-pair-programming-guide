# 14. Как работать в IDE и в терминале

AI-агент живёт либо внутри редактора (IDE), либо в терминале (CLI). Оба подхода рабочие, просто разные по ощущениям. Здесь — типичный рабочий день в каждом.

---

## IDE vs CLI: когда что

| Ситуация | IDE (Cursor, VS Code + Copilot) | CLI (Claude Code, OpenCode) |
|---|---|---|
| Визуальный diff, кнопки, навигация | ✅ | ❌ |
| Большой проект, много файлов | ✅ | ✅ |
| Быстрая правка одного файла | ✅ | ✅ |
| Работа на удалённом сервере (SSH/VDS) | ❌ | ✅ |
| Хочется минимум интерфейса | ❌ | ✅ |

Если не знаешь, с чего начать — бери IDE. Там проще увидеть, что происходит.

---

## Типичный рабочий день в IDE

### 1. Открой проект

`File → Open Folder...` и выбери папку с проектом. Важно: открыта должна быть именно папка проекта, а не отдельный файл.

### 2. Открой чат с агентом

- **Cursor:** `Cmd/Ctrl + L` — боковая панель чата.
- **VS Code + Copilot:** иконка Copilot в боковой панели.
- **Kimi Code / другие плагины:** обычно своя иконка в sidebar.

### 3. Задай задачу

Напиши одно предложение и попроси план:

```text
Нужно добавить валидацию email при регистрации.
Сначала предложи план, потом пишем код.
```

### 4. Смотри diff

Агент предложит изменения. Не жми сразу «Accept all»:

- открой diff (обычно есть вкладка **Diff** или иконка изменений);
- прочитай, что изменилось;
- если агент зацепил лишние файлы — скажи об этом.

### 5. Запусти тесты / проверь вручную

Используй встроенный терминал: `Cmd/Ctrl + `` ` (обратный апостроф).

```bash
npm test
# или
pytest
```

### 6. Закоммить

```bash
git add src/auth/validation.py
git commit -m "feat: add email validation in signup"
```

## Типичный рабочий день в CLI

### 1. Зайди в папку проекта

```bash
cd ~/projects/my-app
```

### 2. Запусти агента

```bash
claude
# или
kimi
# или
opencode
```

### 3. Опиши задачу

```text
Нужно добавить валидацию email при регистрации.
Сначала предложи план, потом пишем код.
```

### 4. Контролируй изменения

Агент будет предлагать правки. Перед согласием:

```bash
git diff
```

### 5. Запусти проверки

```bash
npm test
```

### 6. Закоммить

```bash
git add src/auth/validation.py
git commit -m "feat: add email validation in signup"
```

## Как показать код агенту

Чтобы агент понял контекст, покажи ему конкретные файлы или фрагменты.

### В IDE

- **@file** или **#file** — упомяни файл в чате: `@src/auth/validation.py`.
- **Выдели код мышкой** — многие агенты видят выделенный фрагмент.
- **Скопируй в чат** — для коротких кусков.

### В CLI

- Просто назови файл текстом: «Посмотри `src/auth/validation.py`».
- Для сложных случаев используй `AGENTS.md` в корне проекта — агент прочитает его сам.

## Горячие клавиши, которые стоит знать

| Действие | Windows/Linux | macOS |
|---|---|---|
| Командная палитра | `Ctrl + Shift + P` | `Cmd + Shift + P` |
| Терминал | `` Ctrl + ` `` | `` Cmd + ` `` |
| Найти файл | `Ctrl + P` | `Cmd + P` |
| Найти по проекту | `Ctrl + Shift + F` | `Cmd + Shift + F` |
| Отменить последнее действие | `Ctrl + Z` | `Cmd + Z` |

## Как организовать окно

Удобный минималистичный layout:

- **Слева** — файлы проекта.
- **По центру** — код.
- **Справа** — чат с агентом.
- **Снизу** — терминал.

Ты должен видеть и код, и то, что пишет агент, без постоянного переключения.

---

## Когда переключаться с IDE на CLI

- Работаешь на сервере по SSH.
- Проект очень большой, и IDE тормозит.
- Хочешь быстро что-то проверить без открытия редактора.
- Ты уже хорошо знаешь терминал.

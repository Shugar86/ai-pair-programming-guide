# 15. Что делать, когда ломается

🌐 [English](en/15-troubleshooting.md) · **Русский**

Сломается всегда. Работай по чеклисту — спокойно, шаг за шагом.

---

## До того как паниковать

1. Перечитай ошибку. Часто в первых двух строках написано, что не так.
2. Загугли первые 1–2 строки ошибки. Скорее всего, кто-то уже решил это.
3. Скопируй ошибку и спроси агента: «Вот ошибка. Что она значит и как починить?»
4. Если потратил больше 30 минут без прогресса — позови человека.

**Диалог: расшифровываем ошибку**

```text
Ты: Вот ошибка: `ModuleNotFoundError: No module named 'flask'`. Что это значит?
Агент: Python не находит библиотеку Flask. Скорее всего, она не установлена в текущем окружении.
Ты: Как починить?
Агент: Запусти `pip install flask` или `pip install -r requirements.txt`, если список зависимостей уже есть.
```

## Когда проблема не про окружение, а про агента

Интернет есть, зависимости стоят, git работает — но агент всё равно ведёт себя странно. Чаще всего причина в контексте, а не в коде.

### Агент не понимает проект

**Симптом:** агент предлагает решение, которое не похоже на твой проект. Игнорирует соглашения. Меняет файлы, о которых ты не просил.

**Что делать:**

1. Добавь или обнови `AGENTS.md` в корне проекта. Пропиши стек, conventions, запреты, ответственные. Шаблон — в [`templates/AGENTS.md`](../templates/AGENTS.md).
2. Дай агенту структуру репы: «Сначала прочитай `README.md`, `AGENTS.md` и покажи, какие модули относятся к задаче».
3. Перезапусти сессию, чтобы старый диалог не мешал.
4. Если агент всё ещё путается — начни с одного конкретного файла, а не с общей задачи.

**Куда смотреть:**
- принципы контекста — [`03-principles.md`](03-principles.md);
- настройка сред — [`05-environments.md`](05-environments.md);
- типичные ловушки — [`10-antipatterns.md`](10-antipatterns.md).

### Агент не держит рамки

**Симптом:** пишет много кода, добавляет абстракции, меняет соседние файлы.

**Что делать:**

1. Напомни про минимальный diff и YAGNI — см. [`03-principles.md`](03-principles.md).
2. Попроси план до кода и явно ограничь scope: «Только `src/auth/validation.py`, UI не трогай».
3. Используй чеклисты из [`09-checklists.md`](09-checklists.md) перед приёмкой изменений.

### Агент уверенно врёт

**Симптом:** говорит, что тесты прошли, а они падают. Утверждает, что файл существует, а его нет.

**Что делать:**

1. Не верь на слово. Проверь сам: `git status`, `git diff`, запусти тесты.
2. Попроси агента показать команду и вывод — не просто «всё ок».
3. Если агент не уверен — заставь сказать об этом. См. принцип «Эскалируй неопределённость» в [`03-principles.md`](03-principles.md).

## Git не установлен или не найден

**Ошибка:**

```text
'git' is not recognized as an internal or external command
```

**Что делать:**

1. Скачай git: https://git-scm.com/downloads
2. На Windows во время установки оставь галочку **«Git from the command line and also from 3rd-party software»**.
3. Перезапусти терминал.
4. Проверь: `git --version`.

Если всё ещё не работает — перезагрузи компьютер.

---

## Агент не видит проект

**Симптом:** агент говорит, что не может найти файлы, или предлагает решение, не похожее на твой проект.

**Что делать:**

1. Убедись, что открыта именно папка проекта, а не отдельный файл.
2. В CLI проверь: `pwd` (macOS/Linux) или `cd` (Windows) — должна быть папка проекта.
3. Попроси агента прочитать `README.md` и `AGENTS.md`.
4. Переоткрой проект или перезапусти сессию.

---

## Команда не найдена (command not found)

**Ошибка:**

```text
command not found: node
```

**Что делать:**

1. Убедись, что программа установлена: `node --version`, `python3 --version`.
2. Перезапусти терминал после установки.
3. Проверь PATH. Иногда программа установлена, но терминал её не видит.
4. Переустанови через официальный установщик.

---

## Ошибки при установке пакетов

**Симптом:** `npm install` или `pip install` падает с красным текстом.

**Что делать:**

1. Проверь версию языка: `node --version`, `python3 --version`.
2. Удали кэш и попробуй снова:
   ```bash
   rm -rf node_modules package-lock.json
   npm install
   ```
3. Проверь интернет-соединение и прокси (если работаешь в корпоративной сети).
4. Скопируй полный текст ошибки и покажи агенту.

## WSL — Linux на Windows

WSL позволяет запускать Linux прямо в Windows. Удобно, если проект заточен под Linux/macOS.

### Установка

```powershell
wsl --install
```

После перезагрузки у тебя будет Ubuntu. Открывай его через меню Пуск → Ubuntu.

### Базовые команды

```bash
pwd              # где я сейчас
ls               # список файлов
cd /mnt/c        # попасть на диск C:
```

### Зачем нужен

- Многие инструменты работают в Linux лучше, чем в Windows.
- Терминал в WSL ближе к тому, что используется на серверах.

## SSH — подключение к удалённому серверу

SSH нужен, чтобы заходить на сервер (VDS/VPS) из терминала.

### Подключение по паролю

```bash
ssh username@server-ip
```

### Подключение по ключу

```bash
ssh -i ~/.ssh/my_key username@server-ip
```

### Если не подключается

1. Проверь IP-адрес и порт.
2. Проверь права ключа: `chmod 600 ~/.ssh/my_key`.
3. Убедись, что на сервере разрешён SSH-доступ.
4. Попробуй подключиться с флагом `-v` для подробного вывода: `ssh -v username@server-ip`.

---

## VDS / VPS — удалённый сервер

VDS/VPS — компьютер в интернете, к которому ты подключаешься удалённо. Нужен для деплоя, ботов, серверных приложений.

### Что понадобится

- Аккаунт у провайдера (Hetzner, Timeweb, Selectel и др.).
- SSH-ключ или пароль.
- Базовое понимание Linux.

### После подключения

```bash
ls -la           # посмотреть файлы
sudo apt update  # обновить список пакетов
systemctl status <сервис>  # проверить статус сервиса
```

На проде не экспериментируй без бэкапа. Если не уверен — спроси перед правкой.

## Агент сломал код

**Симптом:** после изменений агента код перестал работать.

**Что делать:**

1. Не паникуй — у тебя есть git.
2. Посмотри diff: `git diff`.
3. Откати конкретный файл: `git checkout -- filename.py`.
4. Или откати весь коммит: `git reset --hard HEAD~1`.
5. Если изменения уже запушены — откати через новый коммит, а не force push.

---

## Ничего не работает: nuclear option

Если проект совсем развалился:

```bash
# Сохранить текущие изменения отдельно
git stash

# Вернуться к последнему рабочему коммиту
git reset --hard HEAD

# Или заново склонировать с GitHub
git clone https://github.com/username/repo.git repo-fresh
```

`git reset --hard` удаляет несохранённые изменения. Используй только если код можно потерять.

## Когда звать человека

- Потратил больше 30 минут и не понимаешь, в чём дело.
- Агент предлагает изменить архитектуру или инфраструктуру.
- Трогаешь прод, базу данных, секреты, CI/CD.
- Видишь ошибки безопасности или подозрительную активность.
